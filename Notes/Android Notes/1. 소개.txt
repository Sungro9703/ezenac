1. 안드로이드의 특징
    1) 핵심 커널 (kernel)
        - 리눅스
    2) 애플리케이션 개발 언어
        - 자바, 코틀린
    3) 오픈소스 지향
    4) 응용 프로그램 프레임워크 (Application Framework)
        - 안드로이드 API가 존재하는 곳
        - 안드로이드 하드웨어에 접근할 때 API를 통해서만 가능

2. 안드로이드 구조
    1) 안드로이드 런타임
        - 자바 코어 라이브러리
        - 아트 런타임 (ART)

3. Hello Android App
    1) Name: 프로젝트 이름
    2) Minimum SDK: 앱이 작동하는 최하 버전

4. 안드로이드 프로젝트 생성
    1) 2개 항목
        - Layout Name: 기본 화면인 XML 파일이름으로 지정 activity_main.xml
        - Activity Name: 기본 소스인 Java 파일 이름으로 지정

    2) 화면 구성을 위한 XML 파일
        - 뷰 컴포넌트 (위젯, 클래스)
            - 부모 뷰, 루트 뷰, 컨테이너 뷰: 다른 뷰를 가질 수 있는 뷰
            - 자식 뷰: 다른 뷰를 가질 수 없는 뷰
        - 레이아웃 뷰
            - 자식 뷰의 배치(위치)를 설정하는 뷰
        
        - 부모 뷰
            자식뷰1
            자식뷰2
            자식뷰3

        - LinearLayout (리니어 레이아웃)
            - 부모가 될 수 있는 뷰 (컨테이너 뷰)
            - 자식의 위치를 설정
                - 가로, 세로
        
        - 크기 설정하는 특수한 키워드
            - match_parent: 부모 뷰의 크기만큼 설정 (가로, 세로)
            - match_content: 내용의 크기만큼 설정 (가로, 세로)

5. 개발환경의 구성
    1) 안드로이드 스튜디오
        - 그레이들(Gradle) 기반 빌드 시스템
        - 레이아웃 에디터
    2) 프로젝트에서 사용되는 폴더 및 파일
        - Java 폴더
            - 자바 소스 파일들이 들어있는 폴더
            - 하위에 패키지 명의 하위 폴더가 존재
                - 프로젝트 생성할 때 입력한 패키지 이름과 동일
        - res 폴더
            - 각종 리소스들이 저장되는 폴더
            - 이미지, 레이아웃, 문자열 등이 들어가는 폴더
            - drawable 폴더: 이미지 파일 저장
            - layout 폴더: 액티비티를 구성하는 xml 파일
            - values 폴더: 문자열 구성하는 string.xml
                           색상표 저장하는 color.xml
                           스타일 저장하는 themes.xml 등 들어있음
        - manifests 폴더
            - AndroidManifest.xml 파일이 들어 있음
            - 앱의 여러가지 정보를 담고 있음
                - 앱의 이름, 컴포넌트 구성과 같은 정보를 가짐
        - Gradle Scripts
            - 그레이들
                - 안드로이드 앱의 빌드 도구
                - 앱을 빌드하는 데 필요한 라이브러리 버전도 자동으로 파악해 필요하다면 다운로드

            - build.gradle(Project) : 전체 프로젝트에 대한 빌드 설정
            - build.gradle(Module: .app) : 빌드 스크립트 핵심 파일,
                                           앱을 빌드 시에 필요한 스크립트

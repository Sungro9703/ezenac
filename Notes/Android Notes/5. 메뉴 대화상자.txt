1. 메뉴의 종류
    1) 옵션 메뉴
    2) 컨텍스트 메뉴
        - 사용자가 화면을 일정 시간 이상으로 길게 누르면 나타나는 메뉴
    3) 팝업 메뉴
        - 액션바에 표시되는 메뉴

2. 옵션 메뉴
    1) 생성 절차
        - 메뉴 XMl 파일 생성
        - 메뉴 파일 등록
        - 메뉴 항목 이벤트 처리
    2) 액션바에 메뉴 표시 옵션
        - showAsAction
            - 액션바에 메뉴 항목 어떻게 나타내느냐 지정
                - always : 항상 보임
                - ifRoom : 액션바에 공간 있는 경우에만 표시
                - never : 메뉴 항목을 오버플로우 영역에 숨김

3. 컨텍스트 메뉴
    1) UI의 어떤 항목과 관련된 동작
    2) 플로팅 컨텍스트 메뉴
        - 사용자가 항목 위에서 오래 누르기 (long click)를 하면 메뉴가 
          대화 상자처럼 표시
        - PC에서 마우스 오른쪽 버튼을 눌렀을 때 나오는 메뉴의 개념과 유사
    3) 생성 절차
        - 컨텍스트 메뉴 등록
        - 컨텍스트 메뉴 생성
        - 메뉴 항목 이벤트 처리

4. 팝업 메뉴
    1) 뷰에 부착된 모달 메뉴 (modal menu)
    2) 용도
        - 오버플로우 스타일 메뉴 제공
        - 서브메뉴 역할
        - 드롭다운 메뉴

5. 대화 상자(Dialog)
    1) 사용자에게 메시지를 출력하고
       사용자로부터 입력을 받아들이는
       아주 보편적인 UI
    2) AlertDialog
        - 제목, 콘텐츠 영역, 액션 버튼
    3) 커스텀 대화상자

6. 알림 기능 (Notification)
    1) 어떤 이벤트가 발생했을 때, 앱이 사용자에게 전달하는 메시지
    2) 알림 구조
        - 작은 아이콘
            - 필수적으로 있어야 함
            - setSmallIcon()으로 설정
        - 앱이름
            - 안드로이드에서 제공
        - 알림 발생시간
            - 안드로이드에서 제공, setWhen()으로 재정의 가능
        - 큰 아이콘
            - 선택 항목, setLargeIcon()로 설정
        - 제목
            - 선택 항목, setContentTitle()로 설정
        - 텍스트
            - 선택항목, setContentText()로 설정
    3) 알림 만드는 절자
        - 알림 채널 생성 (버전8.0)
            - 모든 알림은 채널에 할당되어야 함
            - 채널을 사용하지 않으면 알림이 나타나지 않음
        - 알림 빌더 생성
        - 알림 속성 설정
        - 액션 첨부 (선택)
        - 알림 객체 생성하며 보내기

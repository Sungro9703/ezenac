1. SB 주요 목표
    1) Spring 개발에 대해 빠르고, 광범위하게 적용할 수 있는 환경
    2) 기본값 설정이 있고, 하지만 설정을 변경할 수 있음
    3) XML 구성 요구사항이 전혀 없음
    4) 프로덕션 제품 수준의 스프링 기반 어플리케이션을 쉽게 만들 수 있음

2. 개발환경 구성
    1) JDK 설치
        - Zulu - Azul
        - Temurin - Adoptium
        - Correto - Amazon
        - Oracle OpenJDK - Oracle
        ...
    2) IDE
        - STS
        - IntelliJ IDEA
    3) 주의사항
        - 한글로 작성된 폴더는 피함
        - 설치 관리 용이하도록 특정폴더를 지정
            - 작업공간(workspace)에서 프로젝트를 관리하지 않음
        - 프로젝트 컨텐츠 인코딩 유형은 UTF-8으로 설정
    4) 빌드 도구 (Build Tools)
        - Maven
        - Gradle
            - 코드 컴파일, 배포를 위한 추가적인 작업 수행, 패키징 수행
            - 필요한 라이브러리 관리 (의존성 관리)

3. 소개
    1) 스프링 부트: 스프링 생태계를 기반으로한 개발 플랫폼
    2) 스프링 부트 스타터 (spring-boot-starter) 제공
    3) 단독(stand-alone) 실행 가능한 스프링 애플리케이션 생성
    4) Embedded Tomcat

1. log4j
    1) 로그 기능을 제공하는 오픈 소스 라이브러리
    2) 애플리케이션에서 웹 사이트에 접속한 사용자 정보나 각 클래스의 메서드 호출 시각 등 
       여러가지 정보를 로그로 출력해서 관리 
    3) 메이븐에선 프로젝트 생성시 자동으로 log4j 라이브러리가 설치됨    

2. log4j.xml을 이루는 태그
    1) <appender>
        - 로그를 어디에 출력 할것인지 위치를 결정(콘솔, 파일, db)함.
        - XXXAppender로 끝나는 클래스들의 이름을 보면 출력 위치를 알수 있음.
    2) <layout>
        - 어떤 형식으로 출력할지 출력 레이아웃을 결정함 
    3) <logger>
        - 로깅 메시지를 Appender에 전달함 
        - 개발자가 로그 레벨을 이용해 로그 출력 여부를 조정 할수 있음.  

3. Appender 클래스 
    1) ConsoleAppender
        - 콘솔에 로그 메시지를 출력함 
    2) FileAppender
        - 파일에 로그 메시지를 출력함 
    3) RollingFileAppender
        - 파일 크기가 일정 기준을 넘으면 기존 파일을 백업 파일로 바꾸고 
          처음부터 다시 기록함 
    4) DailyRollingFileAppender
        - 설정한 기간 단위로 새파일을 만들어 로그 메시지를 기록함    

4. PatternLayout 클래스에서 사용하는 출력 속성들
    1) %p 
        - fatal, error, warn, info, debug, trace 등 로그 레벨 이름 출력     
    2) %m
        - 로그 메시지 출력 
    3) %d 
        - 로깅 이벤트 발생 시각 출력 
    4) %F 
        - 로깅이 발생한 프로그램 파일 이름 출력 
    5) %M 
        - 로깅이 발생한 method 이름 출력 
    6) %L 
        - 로깅이 발생한 caller의 라인 수 출력

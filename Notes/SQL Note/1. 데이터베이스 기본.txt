1. 데이터베이스 정의
    1) 특정 조직의 여러 사용자가 공유하여 사용할 수 있도록 통합해서 저장한 운영 데이터의 집합

2. 데이터베이스 사용 이전
    1) 일반적인 텍스트 파일
        - 파일 형태는 여러 사용자가 동시에 공유하기 어려움
        - 파일의 데이터 유실 가능성이 항상 존재
    2) 파일 시스템을 사용
        - 데이터 중복성 문제
        - 업데이트 시 일관성 (consistancy) 유지 어려움
        - 데이터 무결성 (Data integrity constraints) 유지 어려움
        - 데이터 종속성
            - 파일구조가 바뀔 때 마다 응용프로그램 교체 필요
        - 동시성(Concurrency) 제공이 어려움
        - 원자성(atomicity) 제공 어려움
            - 파일 변경 중에 시스템 장애가 발생했을 때 처리 어려움
        - 보안(Security) 제공 이슈
            - 데이터 읽기 권한 제어

3. 모든 것이 데이터베이스로 관리되는 시대
    1) 은행
        - 계좌정보, 입출금 내역 등
    2) 항공사
        - 예약정보, 비행기 스케줄
    3) 대학교
        - 학생정보, 수강신청
    4) 온라인 쇼핑몰
        - 고객기롤, 주문내역
    5) 제조업
        - 제품목록, 주문, 재고, 공급망
    6) 회사
        - 직원정보, 연봉

4. 데이터베이스 특징                                            예)
    1) 실시간 접근 가능                                            목록조회
    2) 계속적으로변화                                              구입정보, 물건재고 정보
    3) 동시 공유가 가능                                            많은 고객이 동시접속, 구매 가능
    4) 저장된 주소가 아닌 내용으로 참조 가능                         가장 많이 팔린 제품은?

5. 데이터의 유형
    1) 정형 데이터 (structured data)
        - 엑셀의 스프레드 시트, 관계
    2) 반정형 데이터 (semi-structured data)
        - HTML, XML, JSON
    3) 비정형 데이터 (unstructured data)
        - 정해진 구조가 없이 저장된 데이터
        - text, 멀티미디어 데이터
